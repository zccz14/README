想要进一步将 CZON 打造成一个在线产品，让用户可以免安装使用，同时又需要降低我们的运维成本。

1. 通过 GitHub Pages 免费托管产品的静态网站，节省服务器成本。使用 Cloudflare 代理流量，使得全球用户都能有较好的访问速度。
2. 使用 GitHub 托管用户数据，节省数据库成本。（带来了一定的复杂性，但是我认为可以接受，否则还需要做很多工作，例如授权系统和整套数据库，且初期用户群体可以接受这种方式。）
3. 通过 GitHub OAuth 获取用户授权，进行文档仓库的读写操作，免 PR，直接 push 到用户仓库的主分支。（等于说用户的数据仓库是完全托管给我们的，但用户可以随时收回去）
4. 产品的静态网站会部署一个在线的 Markdown 编辑器，会加载用户的仓库，用户可以在线编辑文档，然后直接通过 GitHub OAuth 保存到 GitHub 仓库中。
5. 用户需要调用 CZON 的功能时，通过 OAuth OAuth 调用 GitHub Actions 来运行 CZON 的 CLI 功能，将结果保存回用户的仓库中。（这个过程会有一定的延迟，但我认为可以接受，毕竟是免费托管）
6. 用户通过 GitHub Pages 访问他们的文档网站，网站内容直接从 GitHub 仓库中拉取。（这样用户的网站内容也是完全托管给我们的，但用户可以随时收回去）
7. 在 GitHub Actions 中集成 OpenCode 和 其他 AI 服务。需要让用户手动配置 Secret，以便用户能够使用他们自己的 OpenAI Key 或其他 AI 服务 Key。（这样可以避免我们承担高额的 API 调用成本，同时用户也能享受更好的隐私保护，不过目前 OpenCode 有许多免费模型）。
