It is now January 9, 2025, early morning.

I am currently thinking about CZON's custom theme functionality, having previously decided to use JSX to implement this feature.

If we use JSX, the interface becomes URL -> State -> HTML.

The URL remains important because ultimately we need to render static HTML files. For SEO considerations, URLs are still necessary.
Furthermore, the main part of the URL is the Path, not the Query or Hash, as those cannot be translated into static files due to their high dynamic nature.

Therefore, the URL, especially the Path, will still be a core concept in CZON.

React posits that UI = f(State), and JSX is the manifestation of this function f.

So, the problems CZON needs to address are:

1.  Which URLs are accessible?
    1.  All Markdown files. For displaying article content.
    2.  List/index pages, primarily for displaying other URLs under a certain criterion, such as categories, tags, archives, etc.
    3.  Other special pages.

2.  What is the State corresponding to each URL?
    Regardless of the URL type, the State can always be designed to be global.
    Even for article pages, the list of all articles can be passed in, though only the current article is rendered.

3.  How to render HTML based on the State?
    This is the responsibility of JSX. JSX components receive the State as Props, return React Elements, which are then rendered into HTML by ReactDOMServer. If necessary, hydration is performed on the client side.

Thus, CZON's theme system is essentially a set of JSX components that receive the State provided by CZON and render it into HTML... This feels no different from a modern React-based static site generator. ðŸ¤” Can I really compete with already mature solutions?

There are now many mature solutions for SSG (Static Site Generator) that comprehensively use React + Markdown, such as Next.js, Gatsby, Astro, Docusaurus, Nextra, VitePress, etc.
These solutions are already very mature and feature-rich. Why reinvent the wheel?

However, CZON also has its own unique aspects, primarily concentrated in AI integration:

**Native Language Writing, Automatic Translation**

One of CZON's core selling points is supporting content creation in a native language, which is then automatically translated into multiple language versions via AI.
This is not common in existing SSGs. While some plugins can achieve similar functionality, CZON can integrate this more seamlessly.

**Automatic Extraction of Document Metadata**

CZON can automatically extract metadata from documents, such as titles, summaries, keywords, etc., which is very helpful for SEO and content management. The user can act as a writer, with a professional AI assistant tirelessly handling these mundane tasks.

**Intelligent Content Generation and Optimization** (TODO)

Although CZON does not currently have this feature, it could integrate AI in the future to help generate content, optimize SEO, or even dynamically adjust content based on user behavior. For example, AI could review article readability, provide improvement suggestions, or generate related multimedia content (such as images, videos).

**Intelligent Distribution and Recommendations** (TODO)

Content that can better implant itself in the reader's mind is more valuable. All content creators understand this deeply. While creating beautiful frameworks and themes is important, producing content that attracts readers is paramount. CZON needs to consider how to close the loop between content creation and distribution, allowing AI to help authors better reach their audience. For example, generating promotional copy tailored for specific social media platforms with one click, or recommending related articles based on reader interests.

In summary, while CZON overlaps with existing SSGs in some aspects, it has unique advantages in AI integration and content management. Therefore, continuing to develop CZON as an independent tool makes sense. However, it's important to note that CZON should focus on its unique selling points rather than trying to reinvent already mature wheels. For now, functionality is sufficient; perfection is not required.

Consequently, I have decided to temporarily pause the development of CZON's custom theme functionality and instead focus on perfecting its core features, especially the distinctive capabilities in AI integration and content management. This conserves effort. Once CZON's core functionality is more stable, we can reconsider whether to introduce custom theme features or directly integrate with existing SSGs.

In the future, I will write a dedicated article to define CZON's positioning and target user base, clarifying its differences and advantages compared to existing SSGs.
