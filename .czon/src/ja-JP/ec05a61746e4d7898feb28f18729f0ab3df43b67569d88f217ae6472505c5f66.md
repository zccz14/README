今日は要約機能を作成しました。ブログ内の全記事を AI Agent に渡して要約させるものです。このコンテキスト管理は少し面倒だったので、[OpenCode](https://opencode.ai) の Agent を直接統合しました。このツールは MiniMax M2.1 を無料で使用でき、さらに Web サーバーも立ち上げられるので、非常に便利です。

CZON のいくつかの機能を完了しました。

- ダークモード (0.4.3): 自動/手動でのダークモード切り替え、システムテーマに基づく自動切り替えに対応。基本的なレイアウト、画像、Mermaid のダークモード適応をサポート。
- サイトマップ (sitemap.xml) の生成。追加の CLI パラメータ `--baseUrl` が必要です (0.4.2)。
- クローラールール (robots.txt) の生成 (0.4.2)。
- 不要な `-t`, `--template` パラメータを削除 (0.4.2)。

ダークモードにおける一部の特殊なロジック（Mermaid、Tailwind のダークモード適応）は私が手動で調整する必要がありましたが、それ以外の要件は基本的に MiniMax M2.1 が一発でクリアできました。少し驚きました。

CZON に OpenCode を統合して、より深い分析機能を実装する準備を進めています。これにより技術スタックは重くなりますが、OpenCode の Agent 機能を直接利用できるという利点があり、多くの手間を省くことができます。自分で Agent を開発することは、不可能であり、また必要もないでしょう。

さらに、CZON をオンライン製品へと発展させ、ユーザーがインストールなしで利用できるようにしたいと考えています。同時に、運用コストを抑える必要もあります。

1.  GitHub Pages を利用して製品の静的サイトを無料でホスティングし、サーバーコストを削減します。Cloudflare でトラフィックをプロキシ処理し、世界中のユーザーが良好なアクセス速度を得られるようにします。
2.  GitHub を利用してユーザーデータをホスティングし、データベースコストを削減します。（ある程度の複雑さは伴いますが、許容できる範囲だと考えています。そうでなければ、認証システムやデータベース一式など、多くの作業が必要になります。また、初期のユーザー層はこの方法を受け入れてくれるでしょう。）
3.  GitHub OAuth を通じてユーザーの認可を取得し、ドキュメントリポジトリへの読み書き操作を行います。PR を経由せず、ユーザーリポジトリのメインブランチに直接 push します。（つまり、ユーザーのデータリポジトリは完全に当方に委託されますが、ユーザーはいつでも取り戻すことができます。）
4.  製品の静的サイトには、オンライン Markdown エディターをデプロイします。ユーザーのリポジトリを読み込み、ユーザーはオンラインでドキュメントを編集し、GitHub OAuth を通じて直接 GitHub リポジトリに保存できます。
5.  ユーザーが CZON の機能を呼び出す際には、OAuth を通じて GitHub Actions を呼び出し、CZON の CLI 機能を実行して、結果をユーザーのリポジトリに保存し直します。（このプロセスにはある程度の遅延が生じますが、無料ホスティングであることを考えれば許容できる範囲だと考えています。）
6.  ユーザーは GitHub Pages を通じて自分のドキュメントサイトにアクセスします。サイトのコンテンツは GitHub リポジトリから直接取得されます。（これにより、ユーザーのサイトコンテンツも完全に当方に委託されますが、ユーザーはいつでも取り戻すことができます。）
7.  GitHub Actions に OpenCode や他の AI サービスを統合します。ユーザーが自身の OpenAI Key や他の AI サービスの Key を使用できるように、ユーザーに Secret の手動設定を行ってもらう必要があります。（これにより、高額な API 呼び出しコストを当方が負担することを避けられると同時に、ユーザーはより良いプライバシー保護を享受できます。ただし、現在の OpenCode には多くの無料モデルがあります。）
