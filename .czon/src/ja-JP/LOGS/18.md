CZON に OpenCode を統合して、より深い分析機能を実装することを計画しています。これにより技術スタックは重くなりますが、OpenCode の Agent 機能を直接利用できるという利点があり、多くの手間を省くことができます。自分で Agent を開発することは、不可能であり、また必要もないでしょう。

さらに CZON をオンラインプロダクトとして発展させ、ユーザーがインストールなしで利用できるようにしたいと考えています。同時に、運用コストを削減する必要もあります。

1.  GitHub Pages を利用してプロダクトの静的サイトを無料でホストし、サーバーコストを削減します。Cloudflare でトラフィックをプロキシし、世界中のユーザーが良好なアクセス速度を得られるようにします。
2.  GitHub を利用してユーザーデータをホストし、データベースコストを削減します。（ある程度の複雑さは伴いますが、許容範囲内だと考えています。そうでなければ、認証システムやデータベース一式など、多くの作業が必要になります。また、初期のユーザー層はこの方法を受け入れてくれるでしょう。）
3.  GitHub OAuth を通じてユーザーの認可を取得し、ドキュメントリポジトリへの読み書き操作を行います。PR を経由せず、直接ユーザーリポジトリのメインブランチにプッシュします。（つまり、ユーザーのデータリポジトリは完全に当方に委託されますが、ユーザーはいつでも取り戻すことができます）
4.  プロダクトの静的サイトには、オンラインの Markdown エディターをデプロイします。これはユーザーのリポジトリを読み込み、ユーザーがオンラインでドキュメントを編集し、GitHub OAuth を通じて直接 GitHub リポジトリに保存できるようにします。
5.  ユーザーが CZON の機能を呼び出す必要がある場合、OAuth を通じて GitHub Actions を呼び出し、CZON の CLI 機能を実行して、結果をユーザーのリポジトリに保存し直します。（このプロセスにはある程度の遅延が発生しますが、無料ホスティングであることを考えれば許容できると考えています。）
6.  ユーザーは GitHub Pages を通じて自分のドキュメントサイトにアクセスし、サイトのコンテンツは GitHub リポジトリから直接取得されます。（これにより、ユーザーのサイトコンテンツも完全に当方に委託されますが、ユーザーはいつでも取り戻すことができます）
7.  GitHub Actions に OpenCode や他の AI サービスを統合します。ユーザーが自身の OpenAI Key や他の AI サービスの Key を使用できるようにするため、ユーザーに Secret の手動設定を行ってもらう必要があります。（これにより、高額な API 呼び出しコストを当方が負担することを避けられ、同時にユーザーはより良いプライバシー保護を享受できます。ただし、現在の OpenCode には多くの無料モデルがあります。）
