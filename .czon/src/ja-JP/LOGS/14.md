## 影響力について

昨日はDeepSeekのEngram論文のまとめを少しやり、簡単にSNS（朋友圈）にニュースコメントを投稿したところ、いくつかの議論を引き起こしました。皆、大規模言語モデルにはやはり興味があるようです。
私自身は特に新しい見解を生み出したわけではなく、論文の内容をまとめただけです。自分の理解を交え、いくつかの私見を加えました。

これはまさにホットトピックを追うことの利得です。タイムリーにホットな話題に追従し、価値あるコンテンツを生み出せば、皆の注目を集めることができます。もし徐々に自身の影響力を積み上げていければ、将来より多くの機会を得られるでしょう。

私たちNTNLチームは過去に多くの優れたプロジェクトを手がけてきましたが、メンバーが皆控えめで、宣伝が不足していたため、多くのプロジェクトが本来受けるべき注目を得られませんでした。
今後は、より一層宣伝と普及に力を入れ、より多くの人々に私たちのプロジェクトと成果を知ってもらいたいと思います。

将来、私たちの考えや経験を皆さんと共有する機会がより多くあることを願っています。

影響力は長期的に積み上げていくプロセスであり、価値あるコンテンツを持続的に生み出し、積極的にコミュニティと関わっていく必要があります。

最終的には個人のブランドとして現れ、その人が将来より多くの機会とリソースを得る手助けとなります。

これは最終的に私たちがどれだけの結果を手にできるかを決める独立した乗数であり、非常に重要な役割を果たします。

簡単に言えば、あることを行う収益 = (価格設定 - コスト) × 影響力 です。もし影響力が常に一桁のままなら、価格設定がどれだけ高くても、最終的な収益は大きくはなりません。

量的取引を行う際、この影響力という要素はしばしば無視されがちです。戦略さえ良ければ儲かればいい、と。なぜなら、数人の大口顧客にプレゼンするだけで、大口の資金を調達できるからです。
しかし、プロジェクト、特に一般大衆に向けたプロジェクトを行う場合、影響力は非常に重要になります。影響力が大きければ大きいほど、集められる資金とユーザーは多くなり、最終的な収益も高くなります。

とはいえ、結局は同じところにたどり着きます。影響力の大きい人は、往々にしてより良いリソースと機会を持ち、より良いプロジェクトを行うことができ、それが好循環を生み出すのです。

## LegionMindについて

昨日、C1がLegionMindプロジェクトの計画を改善しました。LegionMindは、エージェントのプロジェクト管理ツールとして、GitHub Project Kanban機能を統合することを決定しました。

このインタラクションは明らかにvibe-kanbanに触発されたものですが、vibe-kanbanが独自のカンバンシステムを構築したのに対し、LegionMindはGitHubに元から備わっているProject機能を直接利用します。

こうすることの利点は以下の通りです。

- GitHub Projectは、生産手段（コードリポジトリ）との統合を自然にサポートでき、すべてのタスクとプロジェクトを直接コードに関連付けることができ、管理と追跡が容易です。
- ユーザーは追加のツールを学ぶ必要なく、GitHub Projectの操作看板UIをそのまま直感的に使用できます。さらに、**モバイル端末からのアクセスが可能になります**。これは現在のvibe-kanbanにはない機能です。
- チーム機能がより強力で、**体験がよりシームレスです**。

犠牲になるのは、いくつかのカスタマイズ機能です。例えば、看板のスタイルやレイアウトは、自分で構築したものほど柔軟ではないかもしれません。また、GitHubを使用しないユーザーは対象外となります。

プロジェクトの初期段階では、これらは許容できるトレードオフだと考えます。まずは自分たちにとって最適なワークフローを見つけ、一点を突破してから、より多くのシナリオやユーザーへの対応を考えるべきです。
上記の利点について言えば、vibe-kanbanでも実現可能ですが、追加の開発工数が必要であり、そのために労力を割くのは割に合いません。

最初から何も妥協せず、あらゆる面で完璧で汎用的なものを目指すと、その結果は往々にして両方とも中途半端になり、何も上手くいかなくなるものです。

最近、多くのことを悟りました。物事を行うには取捨選択が必要で、何もかも欲しがってはいけない、と。捨てるものがあってこそ、得るものがあるのです。
