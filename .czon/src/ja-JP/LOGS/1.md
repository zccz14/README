# 解放の曙光が近づいている

現在は2026年1月5日です。

最近、[vibe-kanban](https://www.vibekanban.com/)を使って、エンジニアリングレベルのプロジェクト管理を試してみましたが、なかなか良い効果を感じています。

このツールはカンバンボードを提供し、チームリーダーの視点でタスクと進捗を管理できるようにしてくれます。以前Claude Codeを使っていた時は、自分がシニア開発者で、ジュニア開発者を指導しているような感覚でした。一つのタスクに非常に集中して取り組む必要がありました。Claude Codeにはワークツリーの分離という概念がないため、複数のタスクを同時に管理することができませんでした。
vibe-kanbanを使い始めてからは、タスクを複数のカードに分割し、それらを並行して進めることができるようになりました。各カードにはそれぞれのコンテキストと進捗があり、情報を失う心配なく、いつでも焦点を切り替えることができます。

この方法により、自分が「プログラマー」ではなく、「現場監督」のような存在になったと感じます。具体的なコーディングの詳細に深入りするのではなく、様々なタスクをより良く調整し、リソースを割り当て、進捗を監視することができます。

すぐに、私のボトルネックが新規タスクの作成とタスクのレビュー（そう、明らかにレビューも必要です）に移ったことに気づきました。vibe-kanbanを使っても、不安感は消えませんでした。むしろ、一つのタスクに集中するのではなく、より多くのタスクを管理する必要があるため、その不安はより強くなったとさえ感じます。

一言で言えば、一瞬は快感でしたが、すぐにまた不快になりました。

しかし、明らかにvibe-kanbanには意味があり、使いやすいツールです。ただ、それを使うことで一つの問題に気づかされました。それは、細部への支配欲と、迅速な進捗への欲求との間に矛盾があるということです。そしてこの矛盾は、私の処理能力（バンド幅）の制限によって生じています。以前から存在していた矛盾が、今はより強く感じられるようになっただけなのです。

そこで私は考え始めました。なぜ私はこれほどまでに強い支配欲を持っているのでしょうか？もし細部への支配欲を手放せなければ、真の意味でスケールアップすることはできません。結局のところ、私は有限な人間であり、私の処理能力には限界があります。技術責任者たちについて考えてみると、彼らは自らコードを書くのではなく、管理と調整を通じてプロジェクトを推進しています。彼らはどうやって細部への支配欲を手放す勇気を持てるのでしょうか？単に人を雇うだけでは不十分で、プロジェクトの品質と進捗を確保するためのシステムが必要です。人間とAIという下位の労働力は、本質的には信頼性に大きな違いはありません。人間も間違いを犯しますし、AIも間違いを犯します。重要なのは、全体の信頼性と効率を最大化するシステムをどのように設計するかです。

この問題について詳しく議論し、いくつかの解決策を提案している[この記事](../INSIGHTS/2.md)を参照してください。

また、vibe-kanbanの生態的地位（ニッチ）はClaude Codeとは異なることに気づきました。それはClaude Code / CodeXおよび同種の製品よりも上位に位置します。つまり、Claude Codeたちの進歩は、vibe-kanbanにとって脅威ではなく、進歩をもたらすということです。なぜなら、それらは異なる抽象化レベルにあるからです。Claude Codeたちは下位の労働力であり、vibe-kanbanは管理ツールです。この分業により、それらは競争するのではなく、協調して働くことができます。

しかし、vibe-kanbanは依然としてチームリーダーレベルのツールであり、CEOレベルのツールではありません。CEOが注目すべきは戦略と方向性であり、具体的なタスクや進捗ではありません。したがって、私はタスク管理ツールに依存するだけでなく、戦略的思考能力を高める方法を見つける必要があります。もしかしたら、すでにvibe-okrという製品があるかもしれませんね？😆

仲間と彼が最近取り組んでいるAgentプロジェクトについて話し合いました。彼が以前作った、人間がAIの計画（Plan）をレビューしやすくするツールは面白いと思いましたが、結局のところ解放への道とは逆方向のものだと感じます。彼のツールは最終的に支配欲を強めるものであり、支配欲を手放すものではありません。これは一種の誤りだと思います。真の解放とは、細部への支配欲を手放すことを学ぶことであり、ツールを使って支配欲を強めようとすることではありません。彼らのツールは、よりコントロールしている感覚を与えるかもしれませんが、実際には人間の処理能力を真に解放しているわけではありません。

会社レベル(?) > チームレベル(vibe-kanban) > 個人レベル(claude code, ...etc).

とにかく、まだやるべきことはたくさんありますが、私は実行可能な道筋を見つけたと感じています。
