# 解放の曙光は近づいている

現在は2026年1月5日です。

最近、[vibe-kanban](https://www.vibekanban.com/) を使って、エンジニアリングレベルのプロジェクト管理の試みを始めましたが、なかなか良い効果を感じています。

このツールは、私が Team Leader の視点に立ってタスクと進捗を管理できる看板を提供してくれます。以前 Claude Code を使っていた時は、自分が Senior Developer として、Junior Developer を指導しているような感覚で、常に一つのタスクに高い集中力を注ぎ込む必要がありました。Claude Code には work-tree の分離という概念がなく、複数のタスクを同時に管理する方法がありませんでした。
一方、vibe-kanban を使い始めてからは、タスクを複数のカードに分割し、それらを並行して進めることができます。各カードは独自のコンテキストと進捗を持っており、情報を失う心配なく、いつでも注目する対象を切り替えることができます。

この方法は、自分が「プログラマー」ではなく、「現場監督」のようになったと感じさせます。具体的なコーディングの詳細に深入りするのではなく、各タスクの調整、リソースの割り当て、進捗の監視をより良く行えるようになりました。

しかしすぐに、私のボトルネックは新規タスクの作成とタスクのレビュー（そう、明らかにタスクのレビューも必要です）に移りました。不安感は vibe-kanban を使ったからといって消え去ったわけではありません。むしろ、一つのタスクに集中するのではなく、より多くのタスクを管理する必要が生じたため、その感覚は強くなったとさえ感じます。

一言で言えば、一瞬は快感でしたが、すぐにまた不快感に襲われました。

しかし明らかに、vibe-kanban には意味があり、使いやすいツールです。ただ、それを使うことで、私の細部への制御欲求と迅速な推進への欲求との間に矛盾があることに気づかされました。そしてこの矛盾自体は、私の処理能力（バンド幅）の制限によって生じているものです。以前から存在していたものが、今はより強く意識されるようになっただけなのです。

そこで私は考え始めました。なぜ私はこれほどまでに強い制御欲求を持っているのでしょうか？もし細部への制御欲求を手放せなければ、真の意味でスケールアップすることはできません。結局のところ、私は有限な人間であり、私の処理能力には限界があります。技術責任者たちについて考えてみると、彼らは自らコードを書くのではなく、管理と調整を通じてプロジェクトを推進しています。彼らはどうやって細部への制御欲求を手放す勇気を持てるのでしょうか？単に人を雇うだけでは不十分で、プロジェクトの品質と進捗を確保するためのシステムが必要です。人間とAIという下層の労働力は、本質的には信頼性において大差ありません。人間も間違いを犯しますし、AIも間違いを犯します。重要なのは、全体の信頼性と効率を最大化するシステムを如何に設計するかです。

この問題について詳細に議論し、いくつかの解決策を提案した[この記事](../INSIGHTS/2.md)を参照してください。

また、vibe-kanban の生態的地位（ニッチ）は Claude Code とは異なることに気づきました。それは Claude Code / CodeX および同種の製品よりも、より高い抽象度レベルに位置しています。つまり、Claude Code たちの進歩は、vibe-kanban にとって脅威ではなく、進歩をもたらすものです。なぜなら、それらは異なる抽象度レベルにあるからです。Claude Code たちは下層の労働力であり、vibe-kanban は管理ツールです。この分業により、それらは競争するのではなく、協調して働くことができます。

しかし、vibe-kanban は依然として team-leader レベルのツールであり、CEO レベルのツールではありません。CEO が注視すべきは戦略と方向性であり、具体的なタスクと進捗ではありません。したがって、私はタスク管理ツールに依存するだけでなく、自分の戦略的思考能力を高める方法を見つける必要があります。もしかしたら、すでに vibe-okr という製品があるかもしれませんね？😆

私は仲間と、彼が最近取り組んでいる Agent プロジェクトについて議論しました。彼が以前開発した、人間がAIの計画（Plan）をレビューしやすくするツールは興味深いと思いましたが、結局のところ解放への道筋とは大きく外れていると考えます。彼のツールは最終的に制御欲求を強化するものであり、制御欲求を手放すものではありません。これは一種の誤解だと思います。真の解放とは、細部への制御欲求を手放すことを学ぶことであり、ツールを使って制御欲求を強化しようとすることではありません。彼らのツールは、より強い支配感を人に与えるかもしれませんが、実際には人間の処理能力を真に解放してはいないのです。

会社レベル(?) > チームレベル(vibe-kanban) > 個人レベル(claude code, ...etc).

とにかく、やるべきことはまだたくさんありますが、私は実行可能な道筋を見つけたと感じています。
