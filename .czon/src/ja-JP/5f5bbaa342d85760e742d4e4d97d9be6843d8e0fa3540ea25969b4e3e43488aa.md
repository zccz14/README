# 解放の曙光

現在は2026年1月5日です。

最近、[vibe-kanban](https://www.vibekanban.com/) を使って、エンジニアリングレベルのプロジェクト管理の試みを始めましたが、なかなか良い効果を感じています。

このツールは、私が Team Leader の視点でタスクと進捗を管理できる看板を提供してくれます。以前 Claude Code を使っていた時は、自分が Senior Developer として、Junior Developer を指導しているような感覚で、一つのタスクに高い集中力を注ぎ込む必要がありました。Claude Code には work-tree の分離という概念がないため、複数のタスクを同時に管理することができませんでした。
一方、vibe-kanban を使い始めてからは、タスクを複数のカードに分割し、それらを並行して進めることができるようになりました。各カードは独自のコンテキストと進捗を持っており、情報を失う心配なく、いつでも焦点を切り替えることができます。

この方法は、私を「現場監督」のように感じさせ、「プログラマー」という感覚から遠ざけてくれます。具体的なコーディングの詳細に深入りするのではなく、様々なタスクをより良く調整し、リソースを割り当て、進捗を監視することができるようになりました。

しかしすぐに、私のボトルネックは新規タスクの作成とタスクのレビュー（そう、明らかにレビューも必要です）に移行したことに気づきました。不安感は vibe-kanban を使ったからといって消え去ったわけではありません。むしろ、一つのタスクに集中するのではなく、より多くのタスクを管理する必要が生じたため、より強くなったとさえ感じています。

端的に言えば、一瞬は快感でしたが、すぐにまた不快感に襲われました。

しかし明らかに、vibe-kanban には意味があり、使いやすいツールです。ただ、このツールは私に一つの問題を気づかせてくれました。それは、細部への制御欲求と迅速な推進への欲求との間に矛盾があるということです。そしてこの矛盾自体は、私の処理能力（バンド幅）の制限によって引き起こされています。以前から存在していた矛盾が、今はより強く意識されるようになっただけなのです。

そこで私は考え始めました。なぜ私はこれほど強い制御欲求を持っているのでしょうか？もし細部への制御欲求を手放すことができなければ、真のスケールアップは実現できません。結局のところ、私は有限な人間であり、私の処理能力には限界があります。技術責任者たちについて考えてみると、彼らは自らコードを書くのではなく、管理と調整を通じてプロジェクトを推進しています。彼らはどうやって細部への制御欲求を手放すことができたのでしょうか？単に人を雇うだけでは不十分で、プロジェクトの品質と進捗を確保するためのシステムが必要です。人間とAIという下層の労働力は、本質的には信頼性において大差ありません。人間も間違いを犯しますし、AIも間違いを犯します。重要なのは、全体の信頼性と効率性を最大化するシステムを如何に設計するかです。

この問題について詳細に議論し、いくつかの解決策を提案した[この記事](czon://b283a1863e9af777d581f5af32c0d2d27ed22194f391f5bc3cef9d0d445b4e8c)を参照してください。

また、vibe-kanban の生態的地位（ニッチ）は Claude Code とは異なることに気づきました。それは Claude Code / CodeX および同種の製品よりも、より高い抽象度レベルに位置しています。つまり、Claude Code たちの進歩は、vibe-kanban にとって脅威ではなく、進歩をもたらすものなのです。なぜなら、それらは異なる抽象度レベルにあるからです。Claude Code たちは下層の労働力であり、vibe-kanban は管理ツールです。この分業によって、それらは競争するのではなく、協調して働くことができるのです。

しかし、vibe-kanban は依然として team-leader レベルのツールであり、CEO レベルのツールではありません。CEO が注視すべきは戦略と方向性であり、具体的なタスクや進捗ではありません。したがって、私はタスク管理ツールに依存するだけでなく、自分の戦略的思考能力を高める方法を見つける必要があります。もしかしたら、すでに vibe-okr という製品があるかもしれませんね？😆

仲間と、彼が最近取り組んでいる Agent プロジェクトについて議論しました。彼が以前に開発した、人間がAIの計画（Plan）をレビューしやすくするツールは興味深いものですが、結局のところ解放への道筋とは逆方向を向いていると思います。彼のツールは最終的に制御欲求を強化するものであり、手放すものではありません。これは一種の誤解だと考えています。真の解放とは、細部への制御欲求を手放すことを学ぶことであり、ツールを使って制御欲求を強化しようとすることではありません。彼らのツールは、より強い支配感を人に与えるかもしれませんが、実際には人間の処理能力を真に解放しているわけではないのです。

会社レベル(?) > チームレベル(vibe-kanban) > 個人レベル(claude code, ...etc).

とにかく、まだやるべきことはたくさんありますが、私は実行可能な道筋を見つけられたと感じています。
