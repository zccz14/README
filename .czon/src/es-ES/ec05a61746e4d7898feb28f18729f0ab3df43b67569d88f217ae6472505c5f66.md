Ahora es la noche del domingo 18 de enero de 2026.

Hoy implementé una función de resumen, entregando todos los artículos del blog a un Agente de IA para que los resuma. La gestión de este contexto era un poco complicada, así que integré directamente el Agente de [OpenCode](https://opencode.ai). Esta herramienta permite usar MiniMax M2.1 de forma gratuita e incluso puede levantar un servidor web, ¡es increíblemente útil!
Fue un **cambio radical**, ¡qué diablos con Claude Code, lo descarté directamente!
Volví a trabajar en ello, y este pequeño código es mucho mejor que el actual DeepSeek, aunque claro, cuando salga DeepSeek v4, la cosa podría cambiar.

Completé algunas funciones de CZON.

-   Modo oscuro (0.4.3): Cambio automático/manual al modo oscuro, soporta cambio automático según el tema del sistema, y adaptación básica del modo oscuro para tipografía, imágenes y Mermaid.
-   Generación del mapa del sitio (sitemap.xml), requiere el parámetro CLI adicional --baseUrl (0.4.2)
-   Generación de reglas para rastreadores (robots.txt) (0.4.2)
-   Eliminación del parámetro inútil -t, --template (0.4.2)

Excepto por algunas lógicas especiales en modo oscuro (adaptación para mermaid, tailwind en modo oscuro) que requirieron ajustes manuales míos, básicamente todas las demás necesidades las cumplió MiniMax M2.1 a la primera. Fue una grata sorpresa.

Planeo que CZON integre OpenCode para realizar algunas funciones de análisis más profundas. Esto significa que la pila tecnológica se volverá más pesada, pero la ventaja es poder usar directamente la funcionalidad de Agente de OpenCode, ahorrándome muchos problemas. Desarrollar un Agente propio es realmente imposible e innecesario.

Quiero llevar CZON más allá, transformándolo en un producto en línea que los usuarios puedan usar sin necesidad de instalación, mientras reducimos nuestros costos operativos.

1.  Alojar el sitio web estático del producto gratuitamente en GitHub Pages, ahorrando costos de servidor. Usar Cloudflare para redirigir el tráfico, permitiendo una buena velocidad de acceso para usuarios de todo el mundo.
2.  Usar GitHub para alojar los datos de los usuarios, ahorrando costos de base de datos. (Esto introduce cierta complejidad, pero creo que es aceptable. De lo contrario, habría que hacer mucho trabajo, como un sistema de autorización y toda una base de datos, y el grupo de usuarios inicial puede aceptar este método.)
3.  Obtener autorización del usuario mediante GitHub OAuth para realizar operaciones de lectura/escritura en el repositorio de documentos, sin necesidad de PR, haciendo push directamente a la rama principal del repositorio del usuario. (Es decir, el repositorio de datos del usuario está completamente gestionado por nosotros, pero el usuario puede recuperarlo en cualquier momento).
4.  El sitio web estático del producto desplegará un editor de Markdown en línea que cargará el repositorio del usuario, permitiéndole editar documentos en línea y guardarlos directamente en su repositorio de GitHub mediante GitHub OAuth.
5.  Cuando el usuario necesite invocar funciones de CZON, se llamará a GitHub Actions a través de OAuth para ejecutar las funciones CLI de CZON, guardando los resultados de vuelta en el repositorio del usuario. (Este proceso tendrá cierta latencia, pero creo que es aceptable, ya que es alojamiento gratuito).
6.  Los usuarios accederán a su sitio web de documentación a través de GitHub Pages, y el contenido del sitio se extraerá directamente de su repositorio de GitHub. (Así, el contenido del sitio web del usuario también está completamente gestionado por nosotros, pero el usuario puede recuperarlo en cualquier momento).
7.  Integrar OpenCode y otros servicios de IA en GitHub Actions. Se requerirá que los usuarios configuren manualmente un Secret para poder usar su propia clave de OpenAI o de otros servicios de IA. (Esto evita que nosotros asumamos los altos costos de las llamadas a la API, mientras los usuarios disfrutan de una mejor protección de privacidad, aunque actualmente OpenCode tiene muchos modelos gratuitos).
