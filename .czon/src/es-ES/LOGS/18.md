Quiero llevar CZON aún más lejos para convertirlo en un producto en línea, permitiendo a los usuarios usarlo sin instalación, mientras necesito reducir nuestros costos operativos.

1.  Alojar el sitio web estático del producto de forma gratuita a través de GitHub Pages, ahorrando costos de servidor. Usar Cloudflare para redirigir el tráfico, para que los usuarios de todo el mundo tengan una buena velocidad de acceso.
2.  Usar GitHub para alojar los datos de los usuarios, ahorrando costos de base de datos. (Esto trae cierta complejidad, pero creo que es aceptable; de lo contrario, tendría que hacer mucho trabajo, como un sistema de autorización y toda una base de datos, y el grupo de usuarios inicial puede aceptar este método).
3.  Obtener la autorización del usuario a través de GitHub OAuth para realizar operaciones de lectura/escritura en el repositorio de documentos, sin PR, directamente push a la rama principal del repositorio del usuario. (Es decir, el repositorio de datos del usuario está completamente gestionado por nosotros, pero el usuario puede recuperarlo en cualquier momento).
4.  El sitio web estático del producto desplegará un editor de Markdown en línea, que cargará el repositorio del usuario. El usuario podrá editar documentos en línea y luego guardarlos directamente en el repositorio de GitHub a través de GitHub OAuth.
5.  Cuando el usuario necesite llamar a las funciones de CZON, se llamará a GitHub Actions a través de OAuth para ejecutar las funciones CLI de CZON, guardando los resultados de vuelta en el repositorio del usuario. (Este proceso tendrá cierta latencia, pero creo que es aceptable, ya que es alojamiento gratuito).
6.  El usuario accederá a su sitio web de documentos a través de GitHub Pages, y el contenido del sitio se extraerá directamente del repositorio de GitHub. (De esta manera, el contenido del sitio web del usuario también está completamente gestionado por nosotros, pero el usuario puede recuperarlo en cualquier momento).
7.  Integrar OpenCode y otros servicios de IA en GitHub Actions. Es necesario que el usuario configure manualmente los Secret, para que pueda usar su propia clave de OpenAI u otras claves de servicios de IA. (Esto evita que asumamos altos costos de llamadas a la API, y al mismo tiempo el usuario disfruta de una mejor protección de privacidad, aunque actualmente OpenCode tiene muchos modelos gratuitos).
