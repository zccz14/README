Ahora es la noche del 2026-01-31.

Últimamente he estado jugando mucho con [OpenClaw](https://openclaw.ai), una herramienta de IA muy popular recientemente. Puede controlar toda una máquina con permisos completos. La desplegué en un servidor ECS de Alibaba Cloud, equipado con el modelo MiniMax M2.1, y luego la conecté a mi bot de Feishu. Ahora puedo conversar con OpenClaw a través de Feishu, y OpenClaw me ayuda a ejecutar varias tareas.

Sin embargo, OpenClaw actualmente es más adecuado para instalarse en una máquina local vacía. No es especialmente apropiado para servidores en la nube. Principalmente porque los servidores en la nube son bastante caros. Si OpenClaw no tiene instaladas varias herramientas, como un navegador, su funcionalidad es bastante limitada. Por supuesto, esto también demuestra que las API de este mundo aún no son tan abiertas. A menudo es necesario usar navegadores o varias aplicaciones para completar tareas.

Otro problema es que OpenClaw equipado con MiniMax M2.1 todavía parece no ser muy inteligente. Aunque el autor menciona que Peter Steinberger dijo que MiniMax M2.1 es el mejor modelo de código abierto, también dijo que Opus está muy por delante. En mi opinión, MiniMax M2.1 actualmente sigue siendo bastante débil. No sé cómo sería el rendimiento de Opus + OpenClaw.

Mi suscripción a GitHub Copilot me permite acceder de forma limitada al modelo Opus, pero no puedo usarlo con OpenClaw, solo dentro de OpenCode. Mañana, el 1 de febrero, se reiniciará mi cuota, así que planeo agotarla hoy.

Así que pedí a OpenCode + Opus que me ayudara a escribir el código experimental para la Guerra Prolongada del Capital, [publicado aquí](https://github.com/zccz14/CapitalProtractedWar). Debo decir que intenté esto antes con MiniMax M2.1 y el resultado fue un desastre. Opus funciona notablemente mejor, sin duda el modelo SOTA para programación.

El informe del experimento se mantendrá en el proyecto de código abierto. Pero puedo compartir las conclusiones actuales:

**Bajo un modelo de mercado de alta volatilidad GBM, una estrategia básica de reversión a la media + gestión de capital anti-Martingale puede lograr un crecimiento exponencial del capital incluso en presencia de costos de transacción.**

Sorprendentemente, una estrategia de seguimiento de tendencias con gestión anti-Martingale no logra un crecimiento exponencial, mientras que la estrategia de reversión a la media sí puede. Esto subraya la ventaja de las estrategias con alta tasa de acierto. Sin embargo, todavía tengo dudas al respecto, necesito verificarlo más. Mi intuición me dice que, incluso con una tasa de acierto más baja, una estrategia de seguimiento de tendencias también debería poder lograr un crecimiento exponencial.

Aún se necesita más verificación. ¡Por favor, sigan el proyecto de código abierto del experimento de la Guerra Prolongada del Capital!
