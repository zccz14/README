2026年1月20日，早上。

今天起的不算早，大概 9 点醒了，但是觉得太冷不想起床，于是赖了会儿床，大约到 11 点才真正起床。起来发现下雪了，窗外树上枝头都积了雪，难怪这么冷。
给自己泡了壶热茶，我果然还是更喜欢浓郁的红茶，虽然不是什么名贵的茶叶，但是颜色和味道都很不错。

昨天研究了一下 AI 生成社区评论，见 [普通人版本](../SUMMARY/8-community.md) 和 [原神版本](../SUMMARY/9-genshin.md)（二创人有福了），感觉很有趣。

看评论的时候，人比较轻松，比看正文轻松多了，这就是一个关键的洞察。

为什么很多人宁愿放着正文不看，也要去看评论呢？因为评论更有趣，更轻松，更能引发共鸣。
所以我觉得，未来的内容创作，可能会更多地向这种轻松、有趣的方向发展，而不是一味地追求严肃和专业。

有时候我自己也可以被 AI 生成的评论所触动，这是一个 “啊哈” 时刻，发现 AI 竟然能捕捉到人类情感的细微之处。
或许有的人会评论 AI 生成的内容缺乏灵魂，那不如就让自己做那个点睛之笔。

之后，CZON 的发展路线可以考虑：

1. 评论区：AI + 人类共同打造一个有趣的评论区，吸引更多用户参与讨论。
2. 轻松内容：增加一些轻松、有趣的内容，降低阅读门槛。
3. 用户互动：鼓励用户分享自己的观点和故事，形成社区氛围。

总之，今天的灵感让我对未来的内容创作有了新的思考，期待能在 CZON 上尝试一些新的方向。

现在的自媒体平台愈加封闭，数据难以获取（或者说，合法地获取），他们甚至不愿意用户贴一些外部链接引流。

在商言商，我能理解为什么。但我还是希望走一条去中心化的路线，**用户应该拥有自己的数据和内容**。

比如说，小红书的 Web3 版本？

- **内容托管**：已经有方案了，直接用 GitHub，就算是免费用户也能存储大量内容，远远超过了一般媒体的生产量了，实在不行用户也能自己 Host，随时迁移走。
- **用户身份**：可以搭建一个去中性化的身份系统，例如给钱包地址绑定一个昵称，这样只要连接钱包就能评论。我想这件事大抵还是会做到 EVM 链上，毕竟它的 RPC Readonly 请求不要钱，方便接入。

  再不济可以用 GitHub 账户登录，其实也挺方便的，主要是 CZONE 目前的技术栈已经强耦合到 GitHub 上了，先争取这部分用户吧。

- **移动编辑体验必须对齐主流App**，要让用户跟发小红书、微博和朋友圈一样方便；同时在 PC 端也要有很好的写作体验，毕竟长内容还是在 PC 上写起来舒服。
- **内容推荐和社交网络**：势必需要一个阅读器，它可以索引这个网络中的内容，并且根据用户的兴趣推荐内容。这个可以后续慢慢迭代。层级上应该放入 CZONE，而不是 CZON。

可以想象，这个领域会生成大量的内容和交互，有望打破现有平台的垄断，给用户带来更多选择和自由。

到时候 AI 和人类的内容创作会更加融合、有机。安能辨我是雌雄？太赛博朋克了。不，太图灵了。

作为人间观察而言，确实会是一个非常有趣的实验。

接下来，进一步巩固与 GitHub 的集成和互操作性，积累一定的技术栈，未来完全可以走出一条 GitHub as User Database 的路线。
