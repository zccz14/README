今天是 2026 年 1 月 8 日，晚上。

早上睡的，晚上起来找 Thrimbda 一起聊了一下 ZEN 的未来发展方向，记录一些想法。

他说可以考虑把 ZEN 打造成一个可以发布的知识库，并且表示，如果可以支持自定义前端，他就把自己的博客迁移过来。
具体来说，他的诉求可以拆解成支持自定义主题，支持永久链接，支持文档列表，和支持轮播图。

我觉得这些功能做完以后，确实可以 release 一个 ZEN 1.0.0 版本，或许我该写个发布计划了。

不过我发现 ZEN 的名字已经被用滥了，GitHub 上面有好几个叫 ZEN 的项目，或许我该考虑换个名字。我想要一个比较简短的名字，但是至少在 npm 上面没有被占用的名字，毕竟最终它会使用 npx 命令来启动，CLI 的名字不能太长了。我会把它 host 在我个人域名的子域名下面。倒是也不需要一个独立的二级域名，因为我也没有打算把它做成一个商业化的项目。

搜索了一堆名字，最后觉得 CZone `czon` 还不错，表示 cz + zone (cz zone, content zone)，表示内容的区域，知识的区域。czon 在 npm 上面没有被占用。顺便一提，注册 czone 的时候，npm 说和 clone 太像了，不让注册。但 czon 也不错，还更短了。
