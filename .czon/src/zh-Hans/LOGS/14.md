## 关于影响力

昨天搞了一些 DeepSeek 的 Engram 论文的总结，简单在朋友圈发了个新闻评论，结果引发了一些讨论。看来大家对大语言还是挺感兴趣的。
我本身也没有产出什么新的观点，只是把论文的内容做了个总结而已。结合我自己的理解，加入了我的一些观点。

这其实就是追热点的红利，只要你能及时跟进热点话题，产出一些有价值的内容，就能吸引到大家的关注。如果能逐渐积累起自己的影响力，未来就能有更多的机会。

我们 NTNL 团队过去做过很多优秀的项目，但是因为团队成员都比较低调，缺乏宣传，导致很多项目并没有得到应有的关注。
未来我们会更加注重宣传和推广，让更多人了解我们的项目和成果。

希望未来能有更多的机会，和大家分享我们的想法和经验。

影响力是一个长期积累的过程，需要持续产出有价值的内容，并且积极参与社区互动。

最终会体现在一个人的个人品牌上，帮助他在未来获得更多的机会和资源。

这是最终我们能拿到多大结果的一个独立乘区，有着非常重要的作用。

很简单，做一件事的收益 = (定价 - 成本) x 影响力，如果影响力始终只是个位数，那么无论定价多高，最终的收益也不会太大。

做量化交易的时候，往往会忽略影响力这个因素，觉得只要策略好，赚钱就行了。因为只需要去 pitch 给几个大客户，就能拿到大额资金。
但是做一个项目，尤其是面向大众的项目，影响力就变得非常重要了。影响力越大，能吸引到的资金和用户就越多，最终的收益也就越高。

不过殊途同归，影响力大的人，往往也有更好的资源和机会去做更好的项目，形成一个良性循环。

## 关于 LegionMind

昨天 C1 改进了 LegionMind 项目的计划，LegionMind 决定集成 GitHub Project Kanban 功能，作为 Agent 项目管理工具。

这种交互显然是受到了 vibe-kanban 的启发，不过 vibe-kanban 是自己搭建了一个 kanban 系统，而 LegionMind 则是直接利用了 GitHub 自带的 Project 功能。

这样做的好处是，

- GitHub Project 可以天然支持和生产资料(代码库)的集成，所有的任务和项目都可以直接和代码关联起来，方便管理和追踪。
- 用户可以直接无感地使用 GitHub Project 操作看板 UI，无需额外学习新的工具。而且可以**解锁移动端访问**，这是目前 vibe-kanban 所不具备的功能。
- 团队功能更强，**体验更无感**。

牺牲的是一些自定义功能，比如看板的样式和布局可能不如自己搭建的灵活。而且也放弃了不用 GitHub 的用户。

我认为项目早期，这些都是可以接受的权衡，先找到自己最佳的工作流，以点破面，再考虑适配更多的场景和用户。
要说上面的好处，其实 vibe-kanban 也可以做到，但是需要额外的开发工作量，为了这件事付出，得不偿失。

一开始就什么都不打算妥协，想做的面面俱到通用无比，其结果往往是两头落空，什么都做不好。

最近悟了很多东西，觉得做事要有取舍，不能什么都想要。有舍才有得。
