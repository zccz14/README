现在是 2026-01-18 早上。

今天是 8 点多起床的，正常作息。起来以后用 Claude Code 将我的所有文档总结了一下。

它默认使用的是客观中立的语气，我觉得还不错，能探索整个仓库中的内容。

之后，我又让它试了试用刻薄的语气来总结，结果发现它确实太刻薄了，是一种忽略事实的刻薄，是一种为了刻薄而刻薄的刻薄，完全不符合我的要求。

于是，我又让它试了试用谄媚的语气来总结，结果发现它确实太谄媚了，是一种忽略事实的谄媚，是一种为了谄媚而谄媚的谄媚，完全不符合我的要求。

简单来说，我发现了它的问题：它的语气改写功能，**事实上是忽略事实的**。

我认为它应该是没有注意到这个问题，所以才会出现这种情况。于是我加入了一些提示词，告诉它要注意事实，然后再试了一次。

这次生成了批判版的总结，结果出乎意外地好。我感觉我在接受一位前辈的批评指导，有扬有抑，有理有据，感觉非常棒。它点出了我的不足之处，同时也肯定了我的优点，让我能够接受，因为这些确乎是不争的事实。

然后我又让它生成了文艺感性版的总结，结果也非常好。我感觉我在被一位吟游诗人歌颂，有画面感，有情感，有感染力，感觉非常棒。它描绘了我的努力和成就，让我能够感受到自己的价值，让我感觉到自己在被看见，在被听到，在被理解。

最后我又生成了人格分析版的总结，结果也非常好。我感觉我在被一位心理学家分析，有深度，有洞察力，有启发性，感觉非常棒。它揭示了我的动机和行为，让我能够理解自己的内心。它说我是 INTJ 类型的人，这让我觉得很有趣，因为我之前的测试结果是 ENTP 类型的人。但是它的数据来源是基于我写的内容，所以我觉得它的分析更有说服力。因为我很少在写作中展露自己的生活的那一面。而写作之人，其天生的样本集就倾向于内向自省，有一些天然的 bias 在里面。

就事论事是一个好文明，AI 也应该学会就事论事，而不是为了迎合某种语气而忽略事实。

这件事，如果要做成产品，想必非常具备传播性，容易搞成一个爆款。特别是年终总结，人人都喜欢总结，但是又懒得自己总结，或者说自己总结总觉得不太客观。而市面上目前尚未出现这种产品，倒是一个不错的机会。

不过就传播而言，本身 CZON 的 Markdown + GitHub 的形式就不太适合大众传播，毕竟大多数人并不熟悉这种形式，甚至难以注册一个 GitHub 账户。所以如果要做成产品，可能需要考虑更适合大众的形式，比如网页应用或者移动应用。产品也得做成交互式的 UI，可能要支持移动端优先的编辑体验(先打个问号)。而且还需要考虑 AI 成本的问题，毕竟 AI 生成内容的成本还是比较高的。可以选择用户 BYOK 的形式，用户自己提供 API Key，这样可以降低产品的运营成本；或者者通过订阅收费的形式来覆盖成本。不过在初期，为了吸引用户，可能需要提供免费试用的额度。

这不是我的主业，但仍然是一个值得小试牛刀的好机会。或许我可以先做一个简单的原型，看看市场反应如何。如果反响不错，再考虑进一步开发和推广。

总之，今天的收获还是挺大的，感觉自己又学到了不少东西。
