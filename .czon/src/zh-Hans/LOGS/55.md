预测市场套利项目，名为 PMA = Predict Market Arbitrage。通过一天的 Vibe Coding 已经上线。

我和 Mage 还有 Ryan 三个人用 Opus, GPT, Gemini 一通叮铃桄榔地把代码给写完了。然而，我们三个人都不熟悉 Rust，但是项目是 Rust 写的。

今天用 Opus 4.6 重新做了一遍博客的总结，发现一些问题：

1. Agent 有时候很难保证工作流稳定达到 pass^k 的水平，即 k 次使用均通过。因此一个额外的硬性脚本检查是必要的，并且这个脚本的报错信息要足够友好，能重新提示给 Agent 会话。

   目前，已经在 CZON 的 summary 阶段实现了。

2. 目前，我的博客的所有文章，似乎已经接近 128k token 的限制了。

   一些小的上下文窗口的模型（如 GPT-3.5-turbo-16k）已经无法处理这么大的内容了。

   这件事未来再看怎么解决，等我的博客的内容再多一些再说。

3. 单次写入内容可能过长，可能会导致输出被截断。OpenCode 会无法执行被截断的 write tool 调用。导致无限重试。

   解决方案是先大纲后填充+分段写入。成功写入了一些非常大的总结文章。已经在 CZON 的 summary 命令实现了。

另外，感叹一句 Opus 的 summary 能力真的很强大，思考也很深入。
