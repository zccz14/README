# 解放の曙光が近づいている

最近、[vibe-kanban](https://www.vibekanban.com/) を使って、エンジニアリングレベルのプロジェクト管理を試してみたところ、なかなか良い効果があった。

このツールはカンバンボードを提供し、チームリーダーの視点でタスクと進捗を管理できるようにしてくれる。以前Claude Codeを使っていた時は、自分がシニア開発者で、ジュニア開発者を指導しているような感覚だった。一つのタスクに集中して取り組む必要があった。Claude Codeにはwork-treeの分離という概念がないため、複数のタスクを同時に管理することができなかった。
一方、vibe-kanbanを使い始めてからは、タスクを複数のカードに分割し、それらを並行して進めることができるようになった。各カードは独自のコンテキストと進捗を持ち、情報を失う心配なく、いつでも焦点を切り替えることができる。

この方法は、自分を「プログラマー」ではなく「現場監督」のように感じさせてくれる。具体的なコーディングの詳細に深入りするのではなく、様々なタスクを調整し、リソースを割り当て、進捗を監視することができるようになった。

しかしすぐに、新しいタスクの作成とタスクのレビュー（そう、明らかにレビューも必要だ）がボトルネックになっていることに気づいた。vibe-kanbanを使っても、不安感は消えなかった。むしろ、一つのタスクに集中するのではなく、より多くのタスクを管理する必要があるため、不安は強くなったとさえ感じる。

一言で言えば、一瞬は気持ちよかったが、すぐにまた不快になった。

しかし明らかに、vibe-kanbanには意味があり、使いやすいツールだ。だが、それを使うことで、細部への支配欲と迅速な進捗への欲求の間に矛盾があることに気づかされた。そしてこの矛盾は、私の帯域幅の制限によって引き起こされているものだ。以前から存在していたが、今はより強くなっただけだ。

そこで私は考え始めた。なぜ私はこれほどまでに強い支配欲を持っているのだろう？もし細部への支配欲を手放せなければ、真の意味でスケールアップすることはできない。結局のところ、私は有限な人間であり、帯域幅には限界がある。技術責任者たちについて考えてみると、彼らは自らコードを書くのではなく、管理と調整を通じてプロジェクトを推進している。彼らはどうやって細部への支配欲を手放す勇気を持てるのだろう？単に人を雇うだけでは不十分で、プロジェクトの品質と進捗を確保するためのシステムが必要だ。人間とAIは、本質的には下層の労働力として信頼性に大きな違いはない。人間も間違いを犯すし、AIも間違いを犯す。重要なのは、全体の信頼性と効率を最大化するシステムをどのように設計するかだ。

この問題について詳しく議論し、いくつかの解決策を提案した[この記事](../INSIGHTS/2.md)を参照してほしい。

また、vibe-kanbanの生態的地位はClaude Codeとは異なることに気づいた。それはClaude Code / CodeXおよび同類の製品よりも上位にある。つまり、Claude Codeたちの進歩は、vibe-kanbanにとって脅威ではなく、進歩をもたらす。なぜなら、それらは異なる抽象化レベルにあるからだ。Claude Codeたちは下層の労働力であり、vibe-kanbanは管理ツールだ。この分業により、それらは競争するのではなく、協調して働くことができる。

しかし、vibe-kanbanは依然としてチームリーダーレベルのツールであり、CEOレベルのツールではない。CEOが注目すべきは戦略と方向性であり、具体的なタスクや進捗ではない。したがって、私はタスク管理ツールに依存するだけでなく、戦略的思考能力を高める方法を見つける必要がある。もしかしたら、すでにvibe-okrという製品があるのかもしれない？😆

仲間と彼が最近取り組んでいるAgentプロジェクトについて話し合った。彼が以前作った、人間がAIのプランをレビューしやすくするツールは面白いと思ったが、結局のところ解放への道とは正反対の方向に向かっている。彼のツールは最終的に支配欲を強めるものであり、支配欲を手放すものではない。これは誤解だと思う。真の解放とは、細部への支配欲を手放すことを学ぶことであり、ツールを使って支配欲を強めようとすることではない。彼らのツールは、よりコントロール感を与えるかもしれないが、実際には人間の帯域幅を真に解放しているわけではない。

会社レベル(?) > チームレベル(vibe-kanban) > 個人レベル(claude code, ...etc).

とにかく、まだやるべきことはたくさんあるが、私は実行可能な道を見つけたと感じている。
